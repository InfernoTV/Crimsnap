# GitHub Release Checklist

## Add the secrets

Open:

`https://github.com/InfernoTV/crimsnap/settings/secrets/actions`

Click **New repository secret** and add:

| Name | Value |
| --- | --- |
| `TAURI_SIGNING_PRIVATE_KEY` | Full contents of `src-tauri/updater.key` |
| `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` | Blank for the current generated key, or your key password |

PowerShell helpers:

```powershell
# Print the private key so you can paste it into GitHub.
Get-Content -Raw src-tauri\updater.key

# Confirm the public key embedded in tauri.conf.json matches the local public key.
Get-Content -Raw src-tauri\updater.key.pub
```

Do not commit `src-tauri/updater.key`. It is already in `.gitignore`.

## First publish

```powershell
git init
git add .
git commit -m "Initial public release"
git branch -M main
git remote add origin https://github.com/InfernoTV/crimsnap.git
git push -u origin main
```

Then make the first release:

```powershell
git tag v0.1.0
git push origin v0.1.0
```

The GitHub Action builds and uploads the Windows exe, MSI, setup exe,
signatures, and `latest.json`.

## Updating Later

1. Edit `CHANGELOG.md` and add a new top section, for example:

   ```markdown
   ## 0.2.0

   - Added hotkeys.
   - Fixed overlay selection on high-DPI monitors.
   ```

2. Bump versions:

   - `package.json`
   - `src-tauri/Cargo.toml`
   - `src-tauri/tauri.conf.json`

3. Commit:

   ```powershell
   git add .
   git commit -m "Release v0.2.0"
   git push
   ```

4. Tag the exact same version:

   ```powershell
   git tag v0.2.0
   git push origin v0.2.0
   ```

## How Users Receive Updates

Crimsnap reads:

`https://github.com/InfernoTV/crimsnap/releases/latest/download/latest.json`

That file is generated by `.github/workflows/release.yml` from the installer
signature and the matching `CHANGELOG.md` section. The app checks it on startup
at most once every 6 hours, or immediately when the user clicks
**Settings -> Updates -> Check now**.

If `latest.json` has a newer version and a valid signature, Crimsnap shows the
changelog, downloads the signed installer, runs it passively, then asks the
user to restart.
